Questions (Group One)

Some of the key principals of analysing and designing any website are: Figuring out the websites purpose, simplicity of design for ease of use, a navigation system suited to the website that is not complex, F shaped pattern reading to draw the users eye, visual hierarchy, content, load times no greater than 3 seconds, multi-platform friendly.

To create any website you must also make an architectural plan to do so. Whether you are redesigning an existing website or building a new one you win need to take into consideration the goals of the business and the users it will attract. Once this is done you will need to map out the following – useability, user interaction, user interface design, information design, content strategy and overall website design. There are 4 types of website architectural structures that can be used hierarchical model, sequential model, matrix model and database model.

Design structures are what is used to form the algorithms necessary to design a website they are sequence, selection and repetition. These are used to determine the navigational flow of the website.

User interface design requirements are the guideline you should follow to make your website aesthetically pleasing and easy to use such as – design consistency, responsiveness or ease of access. This should all be used during the production process during the planning and design phase.
